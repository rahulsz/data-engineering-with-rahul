## 🗺️ Overview
Explore core SQL concepts for creating tables, writing queries, using joins, and aggregating data correctly. These queries are foundational for downstream transformations.

## 🧠 Key Concepts
- **DDL / DML** — Data Definition Language (CREATE/DROP) vs Data Manipulation Language (INSERT/UPDATE).
- **JOINs** — Combining rows from two or more tables (INNER, LEFT, RIGHT, FULL).
- **GROUP BY & HAVING** — Aggregating data and filtering post-aggregation.
- **Window Functions** — Performing calculations across a set of table rows related to the current row.
- **CTEs (Common Table Expressions)** — Temporary named result sets to break down complex queries.

## 💻 Code Examples
```sql
-- Creating tables and inserting data
CREATE TABLE shipments (
    shipment_id INT PRIMARY KEY,
    carrier VARCHAR(50),
    cost DECIMAL(10,2),
    delivery_date DATE
);

INSERT INTO shipments VALUES (1, 'FedEx', 15.50, '2025-01-20');
```

```sql
-- CTE and Window Function
WITH RankedShipments AS (
    SELECT 
        carrier,
        cost,
        ROW_NUMBER() OVER(PARTITION BY carrier ORDER BY cost DESC) as rank
    FROM shipments
)
SELECT * FROM RankedShipments WHERE rank = 1;
```

## 🏭 Real-World Application
At an online analytics platform, analysts use SQL to join the 'users' table with the 'page_views' table to calculate the top 10 most active users. Standard GROUP BY aggregates the views, while a Window Function limits the results by region.

## 🎯 Practice Exercises
- 🟢 Beginner: Write a SELECT statement to find all employees hired after 2020.
- 🟡 Intermediate: Write an INNER JOIN between Orders and Customers, calculating the total spent per customer.
- 🔴 Advanced: Write a query using a CTE and the DENSE_RANK() window function to find the second highest paid employee per department.
