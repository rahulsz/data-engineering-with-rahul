## 🗺️ Overview
Distributed data processing using Apache Spark's Python API to tackle Big Data effectively.

## 🧠 Key Concepts
- **Resilient Distributed Datasets (RDDs)** — Spark's fundamental data structure (mostly abstracted away now).
- **Spark DataFrames** — Distributed collection of data organized into named columns.
- **Lazy Evaluation** — Transformations are not executed until an Action (like show or count) is called.
- **Transformations vs Actions** — filter/select (transformations) vs count/collect (actions).
- **Broadcasting & Caching** — Optimizing performance for large joins and iterative algorithms.

## 💻 Code Examples
```python
from pyspark.sql.functions import col, sum, when

# Lazy Evaluation: This defines the transform, but does not execute it
df_filtered = df.filter(col('status') == 'DELIVERED') \
                .withColumn('is_late', when(col('days_taken') > 5, 1).otherwise(0))

# Action: This triggers the execution
late_count = df_filtered.select(sum('is_late')).collect()[0][0]
print(f"Total late deliveries: {late_count}")
```

```python
# Broadcast Join
from pyspark.sql.functions import broadcast

small_lookup_df = spark.read.csv("country_codes.csv")
large_facts_df = spark.read.parquet("user_activity/")

joined_df = large_facts_df.join(broadcast(small_lookup_df), "country_code")
```

## 🏭 Real-World Application
A telecom provider processes 2 TB of call detail records per day. Using PySpark, an engineer distributes the processing across a 50-node cluster, filtering out dropped calls and aggregating total minutes per user using map-reduce paradigms under the hood.
