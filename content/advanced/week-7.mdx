## Overview
Designing scalable data models using Star Schema and the Bronze/Silver/Gold Databricks Medallion approach to structure business intelligence efficiently.

## Key Concepts
- **Bronze Layer** — Raw, unmodified data loaded as-is (often JSON, CSV, or raw Parquet).
- **Silver Layer** — Filtered, cleansed, and augmented data represented as tables.
- **Gold Layer** — Business-level aggregated data, highly refined and modeled (often a Star Schema).
- **Fact vs Dimension Tables** — Facts hold measurable metrics (sales), Dimensions hold context strings (dates, stores).
- **Slowly Changing Dimensions (SCD Type 2)** — Tracking historical changes in dimension records over time.

## Code Examples
```sql
-- Creating a Gold Layer Fact Table
CREATE TABLE gold.fact_sales (
    sale_id BIGINT GENERATED ALWAYS AS IDENTITY,
    date_key INT,
    product_key INT,
    store_key INT,
    quantity INT,
    total_amount DECIMAL(18,2)
) USING DELTA;
```

```sql
-- Implementing SCD Type 2 on a Dimension
MERGE INTO gold.dim_employee AS target
USING silver.employee_updates AS source
ON target.emp_id = source.emp_id 
AND target.is_current = TRUE
WHEN MATCHED AND target.department != source.department THEN
  UPDATE SET is_current = FALSE, valid_to = current_date()
WHEN NOT MATCHED THEN
  INSERT (emp_id, name, department, is_current, valid_from, valid_to)
  VALUES (source.emp_id, source.name, source.department, TRUE, current_date(), NULL);
```

## Real-World Application
A fintech app streams live transaction logs into a Bronze Delta table. The nightly pipeline cleanses timestamps and unifies currency formats into the Silver table, and finally aggregates data into a Gold Star Schema exposing Daily Active Users to the BI team.

## Practice Exercises
- **Beginner**: Draw a rudimentary Star Schema for a library (Books, Members, Authors -> Checkouts fact table).
- **Intermediate**: Write a SQL query to transform JSON strings in a Bronze table into structured columns for a Silver table.
- **Advanced**: Implement an SCD Type 2 pipeline using a PySpark DeltaTable `merge` operation.
